## 1. 解码器与编码器的关系

- 编码器将 \\(4\times5\\) 的输入编码为 memory（编码信息矩阵），shape 仍为 \\(4\times5\\)。
- 解码器负责将 memory 解码为目标序列，模型最终输出依赖解码器。
- 结构：6 个 Encoder Block（每个含多头自注意力、前馈、LayerNorm），6 个 Decoder Block（每个含 2 个多头注意力、前馈、归一化等层）。
- 核心区别：Decoder 每个 Block 有两个多头注意力层（自注意力 + 交叉注意力）。

---

## 2. 解码器输入与机器翻译任务举例

- 解码器输入：
  - \\(Y\\)（目标端输入，含 <start> 符号）
  - 编码信息矩阵 memory
- 例：翻译 "我爱中国" -> "I love China"
  - 源句 "我爱中国" 经嵌入 + 位置编码，得 \\(X\in\mathbb{R}^{4\times5}\\)
  - 目标句 "<start> I love China" 同理，得 \\(Y\in\mathbb{R}^{4\times5}\\)
  - 长度不足补 pad，超长则截断

---

## 3. 解码器结构与注意力机制

- 第一个多头自注意力层输入 \\(Y\\)，输出 \\(4\times5\\)
- 第二个多头注意力层（交叉注意力）：
  - \\(Q\\)：来自上一个自注意力层输出
  - \\(K,V\\)：来自编码器输出 memory
- 交叉注意力实现源-目标信息交互

---

## 4. 解码器输出与最终预测

- 6 个 Decoder Block 后输出 \\(4\times5\\)
- Dense 层：\\(4\times5 \to 4\times5000\\)（假设词表 5000）
- Softmax 层：每个位置选概率最大词，输出最终序列

---

## 5. 训练与推理过程的区别

- 训练：解码器输入完整目标序列（如 <start> I love China），一次性送入网络
- 推理：输入 <start>，逐步生成下一个词，每次拼接已生成词再送入解码器

### 推理流程详解

1. 输入 <start> 和 memory，预测第一个词（如 I）
2. 拼接 <start> I，再输入解码器，预测下一个词（如 love）
3. 依次类推，直到生成 <end> 或达最大长度
4. 每步输出作为下一步输入（自回归生成）

---

## 6. 自注意力矩阵与 mask 机制

### 自注意力矩阵取值规则

- 自注意力层的注意力矩阵为 \\(n\times n\\)，\\(n\\) 为序列长度
- 仅允许 \\(i\\) 位置关注 \\(j\leq i\\) 的词（即只能看自己和前面的词）

|      | <start> | I | love | China |
|------|---------|---|------|-------|
|<start>|   ✓     |   |      |       |
|I      |   ✓     | ✓ |      |       |
|love   |   ✓     | ✓ |  ✓   |       |
|China  |   ✓     | ✓ |  ✓   |   ✓   |

- "✓" 表示允许有注意力，空白为 mask（强制为零）

### mask 的作用

- 保证每个词只能看到自己和前面的词，防止 "偷看" 未来
- 训练时也加 mask，保证行为与推理一致

---

## 7. pad 处理细节

- 输入长度不一需 pad 补齐，pad 位置为无效信息
- 处理方式：
  - pad 位置注意力分数加极大负数（如 \\(-10^9\\)），softmax 后权重趋近 0
  - 有效 token 不关注 pad，pad 之间注意力也被 mask

|      | 你 | 好 | <pad> | <pad> |
|------|----|----|-------|-------|
|你     | ✓  |    |       |       |
|好     | ✓  | ✓  |       |       |
|<pad>  |    |    |  ×    |       |
|<pad>  |    |    |       |  ×    |

- "×" 表示 pad 之间注意力强制为零
- 保证 pad 不干扰有效信息传递和注意力分配

---

## 8. 总结

- Transformer 解码器通过自注意力、交叉注意力和 mask 机制实现高效序列生成
- 推理流程为自回归生成，mask 保证训练/推理一致性
- pad 处理确保变长输入不会影响有效 token 的信息流
