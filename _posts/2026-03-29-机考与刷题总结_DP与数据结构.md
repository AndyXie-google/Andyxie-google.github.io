### 编程1
[a0.09😅😅😅感觉自己真是完蛋了](https://www.nowcoder.com/feed/main/detail/9e2485e1a4984fcca96b0fb292f250a0)
[美团笔试 2026.03.21](https://www.nowcoder.com/discuss/864976447088123904)

#### 题目大意
给定一个数组, 将这个数组复制\\(10^9\\)次得到一个超大的数组, 可以删掉任意数, 但是不能改变排列顺序, 问这个新数组的最长**严格上升**子序列的长度

#### 测试数据
第1行: t, 代表t组测试数据
第2行: n, 代表接下来一组测试数据的长度
第3行: n个数, 范围在\\(0~2*10^5\\)
以此类推
```
2
4
1 1 2 3
5
4 5 3 3 4

----------

3
3
```

#### 解题思路
- 正解: 用一个set去重，然后返回数组的长度就好。(也可以用vector+sort后去重)因为无限拼接的话，必然是可以从最小的数递增到最大的数字的

- 20pts: 动态规划解题, 只能20分(拼接后的数组太长, 即便拼接50次也会栈溢出/错误/超时)
状态转移方程: `ans[arr[i][j]] = max{ans[0]+1, ans[1]+1, ..., ans[arr[i][j]-1]+1, ans[arr[i][j]]}`,
  - `i`: 第i组测试数据
  - `arr[i]`: 第i组测试数据的数组
  - `arr[i][j]`: 第i组测试数据的数组中索引下标为j的项
  - `ans[k]`: 答案数组中当前索引下标中, 末尾项(即升序序列最大项)为k的序列中的最大长度
  - 翻译: 末尾项为`arr[i][j]`的序列中的最大长度, 由末尾项比`arr[i][j]`小的序列中的最大长度+1转移得来

#### 标准答案
```C++
#include <iostream>
#include <vector>
#include <algorithm>
using namespace std;

void solve() {
    int n;
    cin >> n;
    vector<int> a(n);
    for (auto &x : a) {
        cin >> x;
    }
    sort(a.begin(), a.end());
    cout << unique(a.begin(), a.end()) - a.begin() << "\n";
}

int main() {
    std::ios::sync_with_stdio(false);
    std::cin.tie(nullptr);
    std::cout.tie(nullptr);
    int T;
    cin >> T;
    while (T--) {
        solve();
    }

}
```

## 美团2026年春招第三场笔试【硬件开发方向】题解总结
> 考试来源：美团2026年春招第三场笔试（硬件开发方向）

### 编程题1
#### 总结
- 常见报错(segment fault, 段错误)原因
  - 循环终止条件不正确, 导致内存越界访问
  - 变量参数填补错误, 如`arr[i]`写成`arr[n]`, 错误上下文:
  ```C
  int arr[n];
  for(int i = 0; i < n; i++)
  {
     arr[n] //...//
  }
  ```

- C语言整除特性(`/`运算符特性): **对正数除法向下取整, 对负数除法向上取整**
  ```C++
  // n > 0
  floor((double) n / 2) == n / 2

  // n < 0
  ceil((double) n / 2) == n / 2
  ```
p.s: 考试脑子不清楚的时候尽量多调用库函数, 不要自己写向上/向下取整的功能

- 注意做题方法的选择! 没有标签的情况下, 不要着急做题, 想想优化问题到底有没有贪心解法, 再尝试用动态规划解题!

#### 动态规划解法中需要修正的问题

原三维 DP 状态可理解为：
- `dp[i][x][y]`：处理完前 `i` 个数后，还剩 `x` 次操作1、`y` 次操作2 时的最小和。

在这个思路下，主要错误与修正如下。

1. 初始化错误（把不可达状态设成了 0）
- 错误：所有状态默认 0，会让本来不可达的状态参与 `min`，导致答案偏小。
- 修正：用 `INF` 初始化全部状态，仅设置基态为 `dp[0][a][b] = 0`。

2. 转移分支不完整（漏掉单操作转移）
- 错误：当 `x < a && y < b` 时只做了“双操作”转移，`仅操作1` 和 `仅操作2` 被 `else` 吃掉了。
- 修正：三种转移要并列判断：
  - 仅操作1
  - 仅操作2
  - 两种操作都用
  另外还要保留“不操作”。

3. 负数除以 2 的向下取整实现不稳
- 错误：C++ 的 `/` 对负数是向 0 取整，不是向下取整，直接 `x/2` 会错。
- 修正：使用纯整数函数实现 `floor(x/2)`，例如：

```cpp
static inline long long floor_div2(long long x) {
    return x / 2 - ((x < 0) && (x % 2 != 0));
}
```

4. 未判断前驱可达性
- 错误：从不可达状态继续转移会产生伪结果。
- 修正：每个转移前都先判断前驱 `!= INF`。

#### 二、贪心解法要点（100pts）

##### 1. 关键观察

- 操作2：每次都让总和减少 `k`，与选哪个位置无关。
- 操作1：对 `a_i` 的收益是

$$
a_i - \left\lfloor \frac{a_i}{2} \right\rfloor
$$

- 同一位置同时做两种操作时，最终值是

$$
\min\left(\left\lfloor\frac{a_i}{2}\right\rfloor-k,\left\lfloor\frac{a_i-k}{2}\right\rfloor\right)
$$

且恒有

$$
\left\lfloor\frac{a_i-k}{2}\right\rfloor \ge \left\lfloor\frac{a_i}{2}\right\rfloor-k
$$

所以最优顺序可视为“先操作1再操作2”，两者收益可拆开计算。

##### 2. 结论

- 先计算总和 `sum`。
- 计算每个元素做操作1的收益 `gain_i = a_i - floor(a_i/2)`。
- 取最大的前 `a` 个收益相加为 `reduce1`。
- 操作2总收益是 `reduce2 = b * k`。
- 最终答案：

$$
\text{ans} = \sum a_i - \text{reduce1} - b\cdot k
$$

##### 3. 复杂度

- 排序后取前 `a` 个，时间复杂度 `O(n log n)`。
- 空间复杂度 `O(n)`。

该解法可通过大数据范围。

### 编程题2解法要点

目标是从原数组中选一个保持相对顺序的子序列 `b`，最大化

$$
F(b)=\sum_{i=1}^{m}(-1)^{i-1}b_i
$$

即第 1、3、5... 位是加号，第 2、4、6... 位是减号。

#### 1. 状态定义

- `dp_even`：处理到当前位置后，选出偶数个元素时的最大值（包含空子序列，初始为 0）。
- `dp_odd`：处理到当前位置后，选出奇数个元素时的最大值（初始为负无穷）。

#### 2. 转移

遍历每个 `x`：

- `next_odd = max(dp_odd, dp_even + x)`
- `next_even = max(dp_even, dp_odd - x)`

解释：
- 从 `dp_even` 选上 `x`，`x` 处于奇数位，贡献 `+x`。
- 从 `dp_odd` 选上 `x`，`x` 处于偶数位，贡献 `-x`。
- 也可以不选 `x`，所以与原状态取 `max`。

答案是 `max(dp_even, dp_odd)`。

#### 3. 正确性直觉

每个元素只会影响“当前选中长度奇偶性”这一个维度；
因此只需维护奇偶两类最优值，不需要记录完整子序列。
每次转移都覆盖“选/不选”两种决策，最终即全局最优。

#### 4. 复杂度

- 时间复杂度：`O(n)`（每个元素常数次转移）。
- 空间复杂度：`O(1)`（仅维护两个状态值）。

## 牛客网刷题总结
> 题库来源：牛客网刷题（BISHI134 / BISHI135 / BISHI96）

### BISHI134 最大子段和
#### `malloc+memset`
```C++
#include <string.h>
long long* ans = (long long*) malloc(sizeof(long long*) * n);
memset(ans, 0, sizeof(long long)*n);
```

#### `malloc.c:2401: sysmalloc: Assertion`
使用`malloc`分配的内存不够/非法访问未分配的内存(一般是数组越界/指针操作)
```C++
memset(ans, 0, sizeof(long long)*n);
for(int i = 0; i < n-1; i++)    // i < n-1 !!! 注意边界
{
    // ... //
}
```

#### `runtime error/WA`: 想想数据类型是否溢出

#### 解题思路
动态规划: 关键参数是数组尾部索引, 决定了数组是否连续, 但是并不关心数组长度/前面有几项
状态转移方程: `ans[i+1] = max{ans[i]+arr[i+1], arr[i+1])}`, `i`: 当前尾部索引, 即子数组的末尾位置, 选择是否要带上`i+1`项一起求和, 还是从`i+1`项从头开始

### BISHI135 三角形取数(Hard Version)
#### 动态规划
##### 思考顺序
先确定关键物理量, 再想状态转移方程, 最后写for循环具体实现算法; 一定不要先把for循环写出来, 这样不利于思考状态转移方程

##### 基本思路: 局部贪心 -> 全局最优
- 如何取到局部贪心: 有**顺序**的进行局部最优值更新, 一定保证一次总体比较(取max或者取min)能够更新到局部最优, 而没有遗漏可能的情况; 如果在计算后续状态时反过来更新已经前面确定的状态, 一定会错

- ans数组维度的物理含义: 用于确定满足该条件的局部最优. 例如: ans[k]的维度含义如果是: 集合最大元素为k, 那么ans[k]就是"当集合最大元素为k"这个条件满足时的最优答案

##### 如何判断是否能够进行状态压缩
- 首先凭感觉, 真正用于区分状态的参数不能被压缩;

- 然后是根据最后答案的取值方法; 通常最优解有很多个候选值, 他们大多处于同一层; 例如: 最大连续子段和中, 最优解从子段末尾为"0~n"的局部最大字段和中选取, 因此"字段末尾"一定不能被压缩; 同理, 最长严格递增子序列中, 最优解从子序列末尾最大值为0~n的最长序列取得, 因此"末尾值"不能被压缩, 但是当前数组索引(即末尾值在数组中的位置)可以被压缩;

##### 如何得到状态转移方程
有点类似于回溯的思想, 一定是反向推导, 即一个末状态可能通过多少种初状态映射得到; 通常是选取任意的一个中间状态, 尝试搞清楚他的上一状态可能是什么;

##### 初始化和边界问题
- 初始化用宏定义或者有效数组的初值(`arr[0]`)进行初始化, 不要随便初始化为0

- 数据范围存在负数/过大->数组下标越界访问;
  - 如果数组定义在栈空间内, 可能只出现异常值不报错, 也可能会报段错误;
  - 如果使用malloc/vector自动分配, 数组定义在堆空间, 则报堆空间分配错误(一定报错)

- `int->long long`数据类型转换时改变malloc部分的代码, 如果**出现异常输出, 大概率程序本身有越界**, 和malloc无关; 甚至要感谢malloc, 如果用vector自动分配堆空间可能根本发现不了错误; 当然如果实在找不到哪里有问题就用vector.resize/malloc多分配冗余空间, 并且进行正确初始化

- 注意边界检查! 动态规划一旦拿到部分分后很可能状态转移方程本身没有错, 但是在边界情况处理存在问题!

### BISHI96 二叉树先中后序遍历
#### 如何建二叉树
- 输入是 `n-1` 条有向边 `(father, son)`, 表示 `father -> son`。
- 用结构体保存每个节点的左右孩子:
```C++
struct tree {
    int self;
    int l;
    int r;
};
```
- 初始化 `l = r = -1` 表示空孩子。
- 根据题目规则确定左右孩子:
  - 若某节点有两个孩子, 编号小的是左孩子, 大的是右孩子;
  - 若某节点只有一个孩子, 子编号大于父编号则为左孩子, 否则为右孩子。
- 根节点查找方法: 用数组记录“是否作为别人孩子出现过”, 没出现过的节点就是根。

#### 如何用递归写三种遍历
- 先序(根-左-右): 先输出当前节点, 再递归左子树, 再递归右子树。
- 中序(左-根-右): 先递归左子树, 再输出当前节点, 再递归右子树。
- 后序(左-右-根): 先递归左子树, 再递归右子树, 最后输出当前节点。
- 递归写法直观, 代码短, 但当树退化成链且 `n` 很大时有爆栈风险。

#### 如何把递归改成堆栈写法(vector模拟栈)
- 核心: 用 `vector<int> st;` 模拟函数调用栈。
  - 入栈: `st.push_back(x)`
  - 取栈顶: `st.back()`
  - 出栈: `st.pop_back()`

##### 1) 先序的改法
- 递归顺序是根-左-右。
- 栈写法: 弹出即访问当前节点; 为了先处理左子树, 入栈顺序必须是“先右后左”。

##### 2) 中序的改法
- 递归本质是“一路向左到底 -> 回退访问根 -> 转向右子树”。
- 栈写法:
  - `while(cur != -1)` 持续压左链;
  - 左到底后弹栈并访问;
  - `cur = 右孩子` 继续同样流程。

##### 3) 后序的改法
- 难点在于“根必须在右子树访问完后才能输出”。
- 栈写法常用 `last_visited` 记录上次已输出节点:
  - 看栈顶 `u` 时, 若右孩子存在且还没访问过, 转去右子树;
  - 否则说明 `u` 的左右子树都处理完了, 可以输出 `u` 并出栈。

#### 模板化建议
- 同时保留两套函数:
  - 递归版: `first_cal / mid_cal / last_cal`
  - 栈版: `first_order_st / mid_order_st / last_order_st`
- 刷题时先写递归版保证正确性, 再改写为栈版提高大数据稳定性。
