## 要点速览

- 先排查循环边界, 再看算法逻辑, 越界问题通常最先导致失分。
- 极值初始化要避免位运算与优先级陷阱。
- 数据类型与精度匹配直接影响是否 AC。
- 多类型混用场景下, 变量定义错误比公式错误更常见。

## 重点检查`for`循环递增次数过多导致数组越界访问问题
- 现象: `样例不过`, `RA_0~100pts`, `RE_0~100pts`
- 解决办法: 
  - 首先, **如果使用`vector`, 通常不会报`RE`, 所以很难查错, 建议使用`malloc`进行内存分配, 如果出现非法访问会报`RE`**
    
  - 其次尝试自行添加数据集, (通常)交换样例第一行各变量的值(如果该操作合法的话), 即通过改变输入样例第一行输入变量的大小关系, 判断代码中循环边界是否找错了变量

## 最大值/最小值初始化问题
### 什么时候需要初始化?
动态规划(需要对递推起点数值进行初始化), for循环有限次比较求极值, ...

### 初始化方法
```C++
const long long inf_1 = powl(2, 63)-1;  // 最安全, 需要include<math.h>
const long long inf_2 = ((long long)2 << 62)-1; // 2 左移 62 位, 注意不转换为long long 会溢出 
const long long inf_3 = ((long long)1 << 63)-1; // 1 左移 63 位, 移位从1(2^0)开始移, 保证位数和幂指数相等
const long long inf_4 = ((long long)1 << 63);   // 2^63溢出
const long long inf_5 = (long long)1 << 63-1;   // - 运算符优先级高于 << 运算符, 结果为 2^62

cout << inf_1 << " " << inf_2 << " " << inf_3 << " " << inf_4;
// 9223372036854775807 9223372036854775807 9223372036854775807 -9223372036854775808 
```

## 数据格式转换&精度问题
### 数据格式
long double 可以无损转 long long
double 可以无损转 int
float 可以无损转 char

> 注意检查有没有误用导致精度丢失, **特别是题目中存在多种数据格式(`int`, `double`, `long long`, `ull`...)的情况下, 在华为260408实习机考中就出现`double`变量误定义为`long long`, 导致没有`AC`**

## 结论

- C++ 机试稳定性主要取决于边界检查、初始化方式和类型定义。
- 先把越界与类型问题排除, 再优化实现, 通常更高效。
